# Govdirectory Style Guide

This document outlines the style and language guidelines for contributors to the Govdirectory website.

---

## English Variant

When writing content for the website:

1. **Country-specific preference**  
   - If the content is for a specific country that has a preferred English variant, use that variant.  
     - Example: US → American English, UK → British English.  

2. **Default variant**  
   - If there is no country-specific preference, use **British English**.  

3. **Consistency**  
   - Maintain the chosen English variant consistently throughout the page or document.  
   - Document any deviations clearly.  

4. **Common differences**  

| British English | American English |
|-----------------|----------------|
| colour          | color           |
| organise        | organize        |
| licence         | license         |
| centre          | center          |
| travelling      | traveling       |
| cheque          | check           |

---

## Writing Guidelines

- Use **clear, concise, and neutral language**.  
- Avoid jargon; if unavoidable, define it in [`jargon.txt`](./jargon.txt).  
- Use proper **markdown formatting** for headings, code blocks, lists, tables, etc.  
- Always **link to sources** when referencing official government data.  
- Check spelling and grammar with your preferred English variant (default British English).  

---

## Code and Content Formatting

- Use **lowercase for file names** and **snake_case or kebab-case** for URLs.  
- Keep **text content separate** from code; content should be in markdown or template files.  
- Use descriptive **alt text for images**.  
- Ensure **accessibility**: headings, lists, and semantic HTML should be correctly used.

---

## Contributions

- Follow the instructions in [`CONTRIBUTING.md`](./CONTRIBUTING.md) for submitting changes.  
- Create small, focused pull requests with clear commit messages.  
- Always **review changes for consistency** with this style guide.
